top

The 'top' command provides a dynamic real-time view of the system's processes. It's commonly used to monitor CPU and memory usage, and to identify resource-hogging processes.

Purpose: Monitor system performance and active processes.
Usage: Ideal for system administrators and users troubleshooting performance issues.
Application: Use in environments where real-time process monitoring is critical, such as during load testing or system debugging.
Syntax: top
Example: 
$ top

Displays CPU, memory usage, and process details.

kill

The 'kill' command sends a signal to a process, usually to terminate it. If the process doesn't terminate normally, the 'kill' command can send stronger signals to force it to quit.

Purpose: Terminate processes that are unresponsive or consuming excessive resources.
Usage: Commonly used by system administrators to manage runaway or problematic processes.
Application: Useful in troubleshooting, script automation, or during system maintenance to halt specific processes.
Syntax: kill [PID]
Example: 
$ kill 1234

Terminates process with PID 1234.

ping

The 'ping' command is used to test the reachability of a network host. It works by sending ICMP Echo Request packets and waits for responses, helping diagnose network issues.

Purpose: Verify network connectivity and diagnose latency or packet loss.
Usage: Ideal for network troubleshooting and determining the status of remote hosts.
Application: Use in network setup, performance testing, or when debugging internet connection issues.
Syntax: ping [hostname or IP]
Example: 
$ ping google.com

Sends ICMP packets to google.com to test connectivity.

ssh

The 'ssh' command is used to securely log in to a remote system over a network. It provides an encrypted connection for managing servers and performing administrative tasks.

Purpose: Establish secure remote access to systems.
Usage: Widely used by IT professionals for server management and secure file transfers.
Application: Essential for remote server maintenance, cloud management, and development environments.
Syntax: ssh user@hostname
Example: 
$ ssh user@example.com

Connects to the remote server example.com as user.

scp

The 'scp' command is used to securely copy files between hosts on a network using SSH for data transfer. It's a safe way to transfer files to and from remote servers.

Purpose: Transfer files securely over a network.
Usage: Commonly used by developers and administrators for safe data exchange.
Application: Ideal for securely transferring configuration files, backups, and scripts between systems.
Syntax: scp [source] [destination]
Example: 
$ scp file.txt user@remote:/path/to/destination

Copies file.txt to the remote server at the specified path.

curl

The 'curl' command transfers data to or from a server using various protocols (HTTP, FTP, etc.). It is used to download files, interact with APIs, or fetch web pages.

Purpose: Fetch or send data over the internet programmatically.
Usage: Popular among developers for API interaction and web scraping.
Application: Use for downloading files, debugging servers, and automating data retrieval tasks.
Syntax: curl [options] [URL]
Example: 
$ curl https://example.com

Fetches the content of the specified URL.

diff

The 'diff' command compares two files line by line. It shows the differences between them, helping to identify changes between files or directories.

Purpose: Compare files to identify differences.
Usage: Commonly used in version control and patch creation.
Application: Useful for developers to review changes, debug issues, or apply patches.
Syntax: diff [file1] [file2]
Example: 
$ diff file1.txt file2.txt

Shows the differences between file1.txt and file2.txt.

head

The 'head' command shows the first few lines of a file. It's commonly used to preview large files or to check file content quickly.

Purpose: Preview the start of a file.
Usage: Ideal for quickly checking file contents or validating formats.
Application: Use in data analysis, log inspection, or script debugging.
Syntax: head [options] [file]
Example: 
$ head -n 10 file.txt

Displays the first 10 lines of file.txt.

tail

The 'tail' command shows the last few lines of a file. It's often used to monitor log files and check for the most recent entries.

Purpose: View the end of a file, especially for recent changes.
Usage: Commonly used for monitoring logs and real-time data updates.
Application: Ideal for administrators monitoring log files or debugging real-time issues.
Syntax: tail [options] [file]
Example: 
$ tail -n 10 file.txt

Displays the last 10 lines of file.txt.
